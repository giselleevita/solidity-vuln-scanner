Quick Start Guide

1. Install Dependencies

Install Python dependencies (Python 3.10+ required):

pip install -r requirements.txt

Or use the virtual environment:

python3 -m venv venv
source venv/bin/activate # On Windows: venv\Scripts\activate
pip install -r requirements.txt

2. Start the Services

Run the API server and Web UI:

# Terminal 1: Start API
python fastapi_api.py

# Terminal 2: Start Web UI
streamlit run streamlit_ui.py

Or use Docker Compose:

docker-compose up

3. Analyze a Contract

Option A: Using Web UI

Open http://localhost:8501 in your browser and paste your Solidity contract code.

Option B: Using API

curl -X POST "http://localhost:8001/analyze" \
  -H "Content-Type: application/json" \
  -d '{
    "contract_code": "pragma solidity ^0.8.0; contract Test {}",
    "contract_name": "Test",
    "use_llm_audit": false
  }'

Option C: Using CLI

python cli.py contract.sol --llm --format markdown

4. (Optional) Enable AI Features

For AI-powered analysis, add your API key to .env:

USE_LLM=true
LLM_PROVIDER=openai # or "anthropic"
LLM_API_KEY=sk-your-key-here
LLM_MODEL=gpt-4o-mini
Note: The tool works perfectly in "free mode" without an API key - you'll get static pattern analysis only.

5. Run Professional Audit

Option A: Using API

curl -X POST "http://localhost:8001/professional-audit" \
  -H "Content-Type: application/json" \
  -d '{
    "contract_code": "pragma solidity ^0.8.0; contract Test {}",
    "contract_name": "Test",
    "report_format": "json" # or "html", "pdf"
  }'

Option B: Using Web UI

Use the "Professional Audit" tab in the web interface for comprehensive reports.

Example: Analyzing a Vulnerable Contract

Try analyzing this contract with a reentrancy vulnerability:

pragma solidity ^0.8.0;

contract VulnerableVault {
    mapping(address => uint256) balances;
    
    function withdraw(uint256 amount) public {
        require(balances[msg.sender] >= amount);
        (bool success, ) = msg.sender.call{value: amount}("");
        require(success);
        balances[msg.sender] -= amount; // State update after external call!
    }
}

The scanner will detect:

  • CRITICAL Reentrancy vulnerability
  • External call before state update
  • Recommended fix: Use Checks-Effects-Interactions pattern

‚ö†Ô∏è Important Limitations

This tool is for preliminary security assessment and education. It is NOT a substitute for professional security audits before deploying contracts to mainnet.

What the scanner CAN detect:

  • Common vulnerability patterns (reentrancy, access control, etc.)
  • Syntax-level issues
  • Well-known security anti-patterns

What the scanner CANNOT detect:

  • Complex logic-level vulnerabilities requiring domain knowledge
  • Business logic flaws
  • Vulnerabilities in external contract dependencies
  • Protocol-level attacks (flash loans, MEV, etc.)
‚úÖ Best Practice: Always use multiple tools, conduct manual code review, and get professional audits for production contracts handling significant value.
